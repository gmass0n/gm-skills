# Template: `context.md` (single agent-optimized router)

This file is the **entry point** of the map and the most consumed output — `sdd:spec` and `sdd:plan` load it on every run, and each implementer subagent's "task briefing" is distilled from it. It is not dispatched: you assemble it last, mechanically, from the list of files that actually exist in `docs/codebase/` **plus** the enforcer inventory gathered during recon (Step 1).

Two obligations that define this doc:
1. **Lean core** — target **~500 tokens** in the core (invariants + stack + navigation). The catalog can grow with the repo, but the top must fit cheaply in a prompt. If it overflows, you are putting in narrative that belongs in `overview.md` or detail that belongs in the child docs.
2. **Honesty about enforcement** — the "Enforced invariants" section only lists rules tied to a real mechanism (lint/hook/type/interceptor with a path). It is the contract that `sdd:plan`'s `/analyze` checks; an invariant without a mechanism here is a lie that leaks into every future plan.

## Language and shape

English. Paths, library names and symbols verbatim.

## Assembly rules

- **Enforced invariants is the highest-value section.** Each line proves the rule with a mechanism at a real path. No mechanism → not an invariant, it's an aspiration: downgrade it to the separate note or cut it. This section is what `sdd:plan`'s `/analyze` consumes to block an off-pattern plan.
- **List exactly the files that exist.** No dead links, no omissions. Confirm each file in the doc plan (`test -e`) and that each one is linked.
- **Core ≤ ~500 tokens.** Invariants + stack + navigation + per-task loading fit cheaply. The catalog can grow; the core can't.
- The hook describes the doc's specific content ("dependency rule"), not the directory's generic one.
- The end-to-end trace does **not** go here — it goes in `overview.md`. `context.md` is routing + invariants, not narrative.
- `context.md` has no `## Golden rule` — it is a router. It never edits the root `README.md`; it only references it.
